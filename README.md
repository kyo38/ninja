分散DAG実行基盤 実験プロジェクト

■ 概要
--------------------------------------------------
本プロジェクトは、DAG（有向非巡回グラフ）に基づくタスク依存関係を
分散環境で安全に実行するための制御基盤である。

非同期環境における「順序保証問題」を解決するため、
状態同期と依存解決を組み合わせた実行モデルを実装する。

--------------------------------------------------
■ 何が新しいのか
--------------------------------------------------
DAGベース実行と状態同期（state_map + notify）により、
非同期環境でも確実な順序保証を実現する。

■ アーキテクチャ
--------------------------------------------------

          +---------+
          | Client  |
          +----+----+
               |
               v
          +----+----+
          | Master  |
          +----+----+
               |
     -------------------------
     |           |           |
     v           v           v
 +--------+  +--------+  +--------+
 | Worker |  | Worker |  | Worker |
 +--------+  +--------+  +--------+

【通信】
- Client → Master : タスク投入
- Master → Worker : タスク割当
- Worker → Master : 完了通知

--------------------------------------------------
■ タスク定義（例）
--------------------------------------------------

{
  "tasks": [
    { "id": "A", "deps": [] },
    { "id": "B", "deps": [] },
    { "id": "C", "deps": ["A"] },
    { "id": "D", "deps": ["B", "C"] }
  ]
}

--------------------------------------------------
■ 実行モデル
--------------------------------------------------

1. Masterが全タスクを管理
2. 依存関係を解決
3. 実行可能タスクのみWorkerへ割当
4. Workerは完了後に通知
5. state_mapを更新し次タスクを解放

--------------------------------------------------
■ Worker状態遷移
--------------------------------------------------

Idle → Assigned → Running → Completed → Notify

--------------------------------------------------
■ 非同期バグと修正
--------------------------------------------------

【問題】
依存タスクが完了する前に後続タスクが実行される
（フライング実行）

【原因】
非同期処理において完了待ちが保証されていなかった

【修正】
- state_map による状態管理
- Notify による完了通知
- 依存解決ロジックの導入

【結果】
DAG順序保証を実現

--------------------------------------------------
■ 実行手順
--------------------------------------------------

1. リポジトリを取得

   git clone <repo>

2. Masterを起動

   cargo run --bin master

3. Workerを複数起動

   cargo run --bin worker

4. Clientからタスク投入

   cargo run --bin client

--------------------------------------------------
■ 並列実行確認
--------------------------------------------------

- Workerを複数起動することで並列処理を確認可能
- 依存のないタスクは同時実行される
