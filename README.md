SCION to NINJA Roadmap

========================================
Project Overview
========================================
Rustで分散システムおよびSCIONアーキテクチャを理解・実装するためのロードマップ。

最終目標:
- SCIONベース通信の理解
- 分散システム設計の習得
- 実運用可能なRustネットワーク実装

========================================
開発フェーズ
========================================

Phase 1: 分散スケジューリング基盤（完了）
----------------------------------------
達成内容:
- DAGベース実行
- Master / Worker / Client 分離
- TCP通信
- レースコンディション修正

技術的知見:
- タスク送信と完了は非同期
- 状態同期が必須

完了条件:
- DAG順序保証
- 非同期バグ非再現

----------------------------------------

Phase 2: 実行制御強化（進行中）
----------------------------------------
内容:
- リトライ
- タイムアウト
- 並列化

完了条件:
- TaskResultに基づく再試行
- backoff実装
- 並列Worker検証

----------------------------------------

Phase 3: Control Plane
----------------------------------------
内容:
- ノード発見
- 状態監視
- 再スケジューリング

----------------------------------------

Phase 4: セキュリティ
----------------------------------------
内容:
- WireGuard
- PQC（耐量子暗号）
- 鍵管理

----------------------------------------

Phase 5: 実運用
----------------------------------------
内容:
- 組込み
- Fail-safe
- 自動復旧

========================================
フェーズ依存関係
========================================
- Phase2 は Phase1 に依存
- Phase3 は Phase2 に依存
- Phase4 は Control Plane 後

========================================
将来拡張
========================================
- DAG最適化
- 分散化強化
- 観測（ログ・メトリクス）

========================================
システム構成（概念図）
========================================

        +---------+
        | Client  |
        +----+----+
             |
             v
        +----+----+
        | Master  |
        +----+----+
             |
   ---------------------
   |         |         |
   v         v         v
+------+  +------+  +------+
|Worker|  |Worker|  |Worker|
+------+  +------+  +------+

========================================
テスト方法
========================================

1. Master起動
2. Workerを1つ以上起動
3. Clientからタスク送信

期待結果:
- DAG順序で実行される
- Worker追加で並列性が向上する
- エラー時は再試行（Phase2以降）

========================================
最終目標
========================================

分散スケジューラ + SCION統合により、
安全・耐障害・拡張可能な分散システムを実現する
